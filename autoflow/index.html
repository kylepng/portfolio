Auto Flow

Automate bulk AI video & image generation on Google Flow

Auto Flow is a Chrome extension that supercharges Google Flow with batch processing, queue management, multi-resolution downloads, and a smart gallery system. Generate hundreds of AI images and videos with just a few clicks.

Create Image

Generate AI images in bulk using Nano Banana Pro. Enter one prompt per line, or separate multi-line prompts with a blank line.

Entering Prompts

Type or paste your prompts into the text area. Each line is treated as a separate prompt. For multi-line prompts, separate them with a blank line. You can also import prompts from a .txt file.

Reference Images

  • Select images — Persistent reference images used across all jobs. Great for maintaining consistent character styles.
  • One-time upload — Images used only for the current job, then discarded.
  • Bulk mode toggle — When ON, maps one image per prompt in line order. Image 1 pairs with Prompt 1, Image 2 with Prompt 2, etc.

Example Prompt

9:16 portrait mode. pixar 3d animation with subsurface scattering, volumetric lighting, depth of field. A cheerful blue blob character wearing a construction helmet, standing on a microscopic tissue scaffold, arc welding torn muscle fibers back together. no text or labels on image.

Text-to-Video

Generate videos from text prompts in batch. Enter one prompt per line in the text area, select your model and settings, then add to queue.

Model Selection

Model Speed Quality

Veo 3.1 Quality Slower Higher
Veo 2 Moderate Previous gen

Aspect Ratio

Choose between 16:9 (landscape) for YouTube/desktop or 9:16 (portrait) for TikTok/Reels/Shorts.

Repeat Count

Generate N videos per prompt (1–4). Useful for creating variations and picking the best result.

💡

Use 9:16 for TikTok/Reels content. Most viral short-form video is in portrait orientation.

Frame-to-Video

Upload images as starting or ending frames, then Auto Flow animates them into videos using your text prompts. The Frame-to-Video control panel has several input sections.

INPUT Section

Prompt List

  • "Sample .txt file" link — Download a sample prompt file for reference
  • "Import file (.txt)" button — Import prompts from a text file

Queue Images

  • "Select queue images" button — Upload images to the queue (separate from the reference library)
  • "One-time queue upload" button (lightning icon) — Upload images for a single job, then discard them
  • Help button (?) — Tooltip explaining how queue images work
  • Bulk queue mode toggle — "One image per prompt (line order)." When ON, Image 1 pairs with Prompt 1, Image 2 with Prompt 2, etc.
  • Use first prompt for all uploaded queue images toggle — When ON, all queue images use the same first prompt instead of individual prompts

Reference Library

  • "Add to library" button with expand/collapse chevron — Add new reference images to your persistent library
  • Frame hint — "1st image = First Frame, 2nd image = Last Frame (optional)"
  • Library thumbnail grid — Shows all saved reference images as thumbnails
  • Select all / Clear selected / Delete selected refs buttons — Manage your reference library in bulk
  • Selection count — Shows how many reference images are currently selected

Image → Video Workflow

The Image → Video workflow is Auto Flow's most powerful feature. It lets you generate images first, pick the best ones, then animate them into videos — all in one automated pipeline.

1

Write Prompts

Write prompts with [V#-S#] tags and ||| delimiter

2

Generate Images

Run Create Image mode to generate all images

3

Select in Gallery

Browse gallery, pick best images for each scene

4

→ Video / Merge

Send selected images to Frame-to-Video queue

5

Start Queue

Automation uploads images + prompts to Flow

6

Videos Ready

Download finished videos from gallery

Prompt Format

Use this format for each scene — the ||| delimiter separates the image prompt from the video prompt:

[V#-S#] image prompt ||| [V#-S#] video prompt
Scene Tag Image Prompt Delimiter Video Prompt 
— Video 1 (3 scenes) —
[V1-S1] 9:16 portrait. pixar 3d. Blue blob character in construction helmet standing before a torn muscle fiber, dramatic lighting. no text. ||| [V1-S1] Camera slowly pushes in. Blue blob fires up arc welder, sparks fly as it begins repairing the torn tissue. Audio: welding sounds, dramatic score. (no subtitles)
[V1-S2] 9:16 portrait. pixar 3d. Blue blob carefully welding muscle fibers together, glowing repair seam visible. no text. ||| [V1-S2] Close-up of welding in progress. Muscle fibers knit together with golden glow. Blue blob says: "We build different." Audio: sizzling, upbeat. (no subtitles)
[V1-S3] 9:16 portrait. pixar 3d. Blue blob standing proudly on fully repaired muscle, flexing. no text. ||| [V1-S3] Camera pulls back to reveal complete repair. Blue blob flexes victoriously. Audio: triumphant music. (no subtitles)

— Video 2 (3 scenes) —
[V2-S1] 9:16 portrait. pixar 3d. Red-orange blob character with mop bucket approaching inflamed joint. no text. ||| [V2-S1] Red-orange blob rolls up sleeves and dunks mop into anti-inflammatory solution. Audio: sloshing water. (no subtitles)
[V2-S2] 9:16 portrait. pixar 3d. Red-orange blob mopping away red inflammation from joint surface. no text. ||| [V2-S2] Blob mops vigorously. Red inflammation dissolves into clean tissue underneath. Audio: satisfying cleaning sounds. (no subtitles)
[V2-S3] 9:16 portrait. pixar 3d. Red-orange blob wringing out dirty mop, joint now clean and healthy. no text. ||| [V2-S3] Final squeeze of the mop. Joint gleams. Red-orange blob nods approvingly. Audio: sparkling clean chime. (no subtitles)
⚠️

Key Rules

  • Each VIDEO gets a unique V# (V1, V2, V3...) — do NOT use V1 for everything
  • The [V#-S#] tag must appear on BOTH sides of the ||| delimiter
  • Keep one scene per line
  • Separate video groups with a blank line for readability
  • The image prompt describes what to GENERATE (still image)
  • The video prompt describes how to ANIMATE it (motion, audio, dialogue)

Gallery

Your generated content is automatically collected in the gallery. Browse, select, download, and send images directly to the video pipeline.

The gallery has two main tabs — Images and Videos — each with their own set of controls for browsing, selecting, and exporting your generated content.

Image Gallery

The Image Gallery tab displays all generated images in a browsable grid. Use the toolbar controls to manage your content:

Toolbar Controls

  • Images / Videos sub-tabs — Switch between viewing generated images and generated videos
  • Layout toggle — Switch between Default (compact grid) and Large (bigger thumbnails) views
  • Resolution dropdown (1K) — Select the resolution for scanning and display
  • Scan All (yellow/highlighted button) — Scans the Google Flow page for all generated images. This is the primary way to populate the gallery.
  • Import Images — Import images from your local computer into the gallery
  • Sort dropdown (Default) — Sort images by different criteria
  • Image count — Displays the total number of images found

Selection & Actions

  • Click images to select them (checkmark overlay appears)
  • Random Pick — Randomly select images from the gallery. Useful for quick variety when you have many generated images.
  • Pick Matched — Automatically select the best-matched images for each prompt. Smart selection based on prompt-image pairing.
  • Deselect All — Clear all current selections
  • Download (N) — Download all selected images at the chosen resolution (1K instant, 2K/4K upscaled)
  • Animate Images — Send selected images to the Frame-to-Video queue for animation. This is the key Image-to-Video bridge that connects the image pipeline to video generation.
  • "How to use" expandable section — Built-in guide within the gallery for quick reference

Video Gallery

  • Videos are grouped by prompt for easy browsing
  • Layout controls — Same Default/Large layout toggle as the image gallery
  • Download at 720p (instant) or 1080p/4K (serialized upscale)
  • Selection & batch download — Select multiple videos and download them all at once

Gallery Workflow

  1. Generate images or videos using any mode (Create Image, Text-to-Video, etc.)
  2. Switch to the Gallery tab
  3. Click "Scan All" to populate the gallery with your generated content
  4. Browse and select the best results
  5. Download at your desired resolution, OR
  6. Click "Animate Images" to send selected images to the video pipeline
💡

4K video downloads are serialized — only one upscale processes at a time. Be patient!

Queue Management

The queue system lets you stack multiple generation jobs and run them back to back. Monitor progress, reorder priorities, and handle failures gracefully.

Adding Jobs

Select your mode, enter prompts, configure settings, then click "Add to Queue". Jobs are added in order and can be rearranged before starting.

QUEUE Section

  • Job Download Folder — Text input with auto-incrementing folder name (e.g., "Auto-Flow-08"). Sets where the output for each job is saved.
  • Queue manage button — Opens the full queue modal to view, reorder, and manage all jobs
  • Auto-start next job in queue — Toggle to automatically begin the next job when the current one completes

PROGRESS Section

  • Progress bar — Visual progress indicator showing completion percentage
  • Status — Shows current state (Ready, Running, etc.) with a checkmark icon when complete

Queue Modal

Open the queue modal to view all jobs and their current status:

Pending Running Done Failed

Actions

  • Move to front — Prioritize a job to run next
  • Reset — Re-run a completed job from scratch
  • Delete — Remove a job from the queue
  • Redo selected — Pick specific prompts within a job to retry
  • Inline settings — Edit model, aspect ratio, and repeat count before running

Failed Tasks

  • Failed count — Shows the number of tasks that have failed
  • Placeholder — Shows "No failed prompts yet." when there are no failures
  • Copy Failed Prompts/Images button — Copies all failed prompts to your clipboard for quick re-entry
  • Retry Failed button — Creates a new job from all failed prompts and adds it to the queue
  • Auto retry failed until 0 checkbox — Automatic retry loop that keeps retrying failed prompts until none remain. Maximum 12 retry rounds.
💡

The "Auto retry failed until 0" feature is great for overnight batch runs. It will keep retrying failed prompts automatically (up to 12 rounds) so you don't have to babysit the queue.

Settings

Fine-tune Auto Flow's behavior to match your workflow. Access settings from the gear icon in the control panel.

General Settings

  • Videos per task — How many videos to generate per prompt (dropdown: 1–4)
  • Video Model — Veo 3.1 Fast (Lower Priority), Veo 3.1 Quality, or Veo 2
  • Video Ratio — Landscape (16:9) or Portrait (9:16)
  • Images per task — How many images to generate per prompt (dropdown: 1–4)
  • Image Model — Nano Banana Pro (Default), or other available models
  • Image Ratio — Landscape (16:9) or Portrait (9:16)
  • Start from (Prompt/Image) — Skip to a specific prompt number. Useful for resuming interrupted jobs.

Timing

  • Video creation wait time (min/max seconds) — Random wait between min and max after each generation
  • Humanized mode (experimental) — Adds random delays to simulate human browsing behavior
  • Typing speed (humanized) — Slider from 0.5x to ~3.0x controlling how fast prompts are typed. Renamed from "Humanized speed" for clarity.

Download Settings

  • Auto-download videos — Toggle on/off + "Configure folder" link to set download destination
  • Video resolution — Original (720p), 1080p, or 4K
  • Auto-download images — Toggle on/off + resolution dropdown (1K, 2K, 4K)
  • Image resolution — 1K, 2K, or 4K
💡

Turn off "Ask where to save each file before downloading" in your browser's download settings for seamless auto-downloading.

Tips & Shortcuts

Power-user tips to get the most out of Auto Flow.

  • Use bulk mode for efficient image-per-prompt batching
  • Set up reference images for consistent character style across scenes
  • Queue multiple jobs before pressing Start — they'll run back to back
  • Enable auto-download + auto-start for fully hands-off operation
  • Use [V#-S#] tags to keep multi-scene videos organized
  • Lower the humanized speed multiplier if you're getting rate limited
  • The Gallery persists across queue restarts — your generated content won't disappear
  • Use 9:16 portrait for TikTok, Reels, and Shorts content
  • Enable Scan in the gallery if content isn't appearing after generation
  • Redo selected lets you retry specific prompts without re-running the whole job
